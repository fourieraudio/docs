# Fourier Audio transform.engine v1.6.16 Release Notes 

* **transform**.engine **Version**: 1.6.16  
* **Windows transform**.client **Version**: 1.6.16  
* **macOS transform**.client **Version**: 1.6.16

## Summary

**transform**.engine v1.6.16 is a maintenance release targeting a plugin compatibility issue introduced in v1.6.15, where certain plugins could crash or spike. This release changes the graphics driver version used within **transform**.engine, resolving these plugin compatibility issues.

## Bug Fixed In This Release

- **TF-4742**: Graphics driver issue introduced in v1.6.15 could cause certain plugins to crash.

## Compatibility Notes

Showfiles are fully backwards and forwards-compatible with **transform**.engine v1.1 \- v1.6.16.

**transform**.engine v1.6.16 is fully backwards compatible with **transform**.client v1.0-v1.6.15, and vice-versa; you can connect to an engine running v1.5 from a client running v1.6, and to an engine running v1.6 from a client running v1.5.

Downgrading to versions prior to v1.6.16 is not possible once v1.6.16 has been installed.

## Known Issues

Users should be aware of the following known issues in this release, which we hope to address in a future update:

- **TF-2749**: A bug in macOS 15.2-15.4.1 causes macOS to block “Local Network” access for apps after reboot until manually enabled again in Settings. We can confirm that this is fixed in macOS 15.6.1.  
- **TF-2698**: When bx\_limiter and AMEK EQ 200 are used in a chain, recalling cues can cause a momentary increase in DSP load. We are working with the plugin publisher to resolve this issue.  
- **TF-3434**: It is possible for multiple network interfaces to be configured with overlapping subnets via DHCP, causing IP network issues.

A small number of plugins may experience compatibility issues running on **transform**.engine. For details, see our plugin compatibility database at [https://plugins.fourieraudio.com](https://plugins.fourieraudio.com) .
